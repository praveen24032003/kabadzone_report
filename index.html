Section 01
About the Project

Kabadzone is a professional digital platform built to manage every aspect of kabaddi tournament operations — from player registration and team formation to live match scoring and public viewing.

This is not a simple scoreboard application. Kabadzone is a full sports operations platform designed to become the digital backbone for kabaddi tournament administration and live engagement, serving organisers, officials, players, and fans alike.

The platform provides:

  • A mobile application for players, team managers, scorers, and the public
  • A web-based administration portal for super administrators
  • A real-time live scoring system that records every match event
  • Approval-based governance for player registrations and tournament creation
  • Notifications, reports, audit trails, and district-level administration
Section 02
What Does the Platform Do?
P
Player Management
  • Players register via Google sign-in and complete a verified profile
  • District administrators review and approve or reject player registrations
  • Players can view their stats, joined teams, and match history
T
Team Management
  • Team managers create teams and build rosters
  • Managers can invite players or approve join requests
  • Teams register for tournaments through the platform
E
Tournament Management
  • Committees create tournament events with dates, locations, and rules
  • District administrators review and approve tournament proposals
  • Fixtures and match schedules are managed within each tournament
L
Live Match Scoring
  • Assigned scorers record every match event in real time — points, cards, timeouts, substitutions, and official decisions
  • The system calculates live scores automatically from event data
  • Public viewers see scores update instantly through real-time connections
  • Full match history is preserved for review and audit
A
Administration & Reporting
  • Super administrators have a global dashboard with full platform oversight
  • District administrators manage operations within their assigned regions
  • Audit logs track all sensitive actions for accountability
  • Reports provide summaries at system, district, and tournament levels
N
Notifications
  • In-app notification inbox for approvals, invitations, and updates
  • Push notifications delivered to registered mobile devices
Section 03
Who Uses the Platform?

The platform supports seven distinct user roles, each with controlled access permissions:

Role Description
Super Admin Full platform oversight, global settings, all reports, and system governance
District Admin District-level approvals for players and tournaments, regional moderation and reports
Committee Tournament creation, match scheduling, and scorer assignment
Scorer / Umpire Live match scoring, event recording, and officiating for assigned matches only
Team Manager Team creation, roster management, player invitations, and tournament registration
Player Player profile completion, team participation, personal stats, and invitation responses
Public Viewer Live match viewing, tournament discovery, leaderboards, and statistics

Each role has clearly defined permissions. For example, a District Admin can only approve players and tournaments within their assigned district, while a Scorer can only operate on matches specifically assigned to them.

Section 05
Databases & Storage

The platform uses three distinct data systems, each serving a specific and clearly defined purpose:

P
PostgreSQL
Hosted on Supabase / Amazon RDS
Primary Database — Single Source of Truth

PostgreSQL is the main database that permanently stores all platform data. Every record that must be accurate, permanent, and auditable lives here.

  • User accounts and login sessions
  • Player profiles and approval records
  • Team details and roster memberships
  • Tournament information and registrations
  • Match schedules and scorer assignments
  • Every scoring event recorded during a live match (points, cards, timeouts, substitutions, official decisions)
  • Live score totals derived from match events
  • Notifications with read/unread status
  • Audit logs for security tracking and accountability
  • Invitations, join requests, and push notification delivery records
R
Redis
Hosted on Upstash / Amazon ElastiCache
Speed & Real-Time Layer

Redis is an ultra-fast temporary data system that does not store permanent data. It acts as a speed booster and real-time helper.

  • Live score broadcasting — pushes match updates to thousands of viewers instantly
  • Caching frequently accessed data so the app responds faster
  • Rate limiting to prevent system abuse
  • Session revocation support when a user logs out or is blocked
  • Short-lived coordination data for real-time operations
F
Firebase
Google Cloud Platform
Identity, Media & Notification Layer

Firebase works alongside PostgreSQL. PostgreSQL remains the master record; Firebase handles identity verification, notification delivery, and file storage.

  • Firebase Auth — verifies that users are who they claim to be during Google sign-in
  • Firebase Cloud Messaging (FCM) — delivers push notifications to Android and iOS devices
  • Firebase Storage — stores uploaded images such as player photos, team logos, and tournament banners

Section 09
Risks & Mitigation
Risk Mitigation Strategy
Live scoring must handle high traffic during matches Event-based architecture ensures every action is recorded individually, and scores are calculated server-side for accuracy under load
Multiple user roles increase security complexity Role-based access control is enforced at the server level on every request, not just in the application interface
Real-time updates must be reliable for public viewers WebSocket connections with reconnection support ensure viewers stay connected and can rejoin without data loss
Data integrity is critical for tournament records PostgreSQL as the single source of truth with audit logging for all sensitive operations
Mobile app must work across Android and iOS React Native with Expo provides a single codebase that runs natively on both platforms
